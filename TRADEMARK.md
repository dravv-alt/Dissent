# Trademark Policy

## Project Identity

**Dissent** is the official name of the open-source AI behavioral auditing framework
originally created by Dravvya Jain and hosted at:

> https://github.com/dravv-alt/Dissent

---

## What Is Open

The source code of Dissent is open-source, released under the
[Apache License 2.0](LICENSE). This means:

- ✅ **Forks are allowed.** You may fork this repository and build upon it.
- ✅ **Modifications are allowed.** You may modify the source code freely.
- ✅ **Commercial use is allowed.** You may use Dissent in commercial products and services.
- ✅ **Redistribution is allowed.** You may distribute modified or unmodified copies.
- ✅ **Private use is allowed.** You may use Dissent privately without disclosure.
- ✅ **Contributions are welcome.** See [CONTRIBUTING.md](CONTRIBUTING.md).

---

## Identity Protection

While the code is fully open, the **Dissent** project name and identity are protected
to prevent confusion about which project is the original.

The following uses of the name **"Dissent"** are **not permitted** without explicit written
permission from the original author:

- ❌ Naming a fork or derivative work **"Dissent"** in a way that implies it is the
  official or original project.
- ❌ Using "Dissent" in product names, domain names, or marketing materials in a way
  that could mislead users into believing they are using the original project when
  they are not.
- ❌ Representing a fork or derivative work as officially affiliated with, endorsed by,
  or maintained by the original Dissent project or its author.

---

## Fork Branding Guidance

If you fork Dissent and make substantial modifications, we encourage you to:

1. **Adopt your own project name** — one that is clearly distinct from "Dissent."
2. **Retain the NOTICE file** — as required by Apache 2.0, clearly acknowledging
   the original project and its author.
3. **Link back to the original repository** where appropriate.

You are free to describe your project as **"based on Dissent"** or **"derived from Dissent"**
as long as it is clear your project is a fork and not the original.

---

## Attribution Requirements

All forks and derivative works must retain:

- The [NOTICE](NOTICE) file (required by Apache 2.0).
- The [LICENSE](LICENSE) file.
- A reference to the original repository: https://github.com/dravv-alt/Dissent

---

## Purpose of This Policy

This policy exists to protect **project identity and attribution**, not to restrict
open-source activity. The intent is:

> Anyone should be able to build on Dissent, contribute to Dissent,
> fork Dissent, and extend Dissent.

> The original project, original repository, original architecture,
> and original author must always be clearly acknowledged.

This is an **identity protection policy**, not an anti-fork policy.
